# Ferrum MinimalDemo Sample

This sample demonstrates a complete end-to-end .NET MAUI app using Ferrum to call native C code on iOS and Android.

The Ferrum framework includes multiple native test fixtures to validate different patterns:
- **test_stub**: Simple `ferrum_add(int, int)` function
- **dsp_fixture**: Float buffer processing (`ferrum_dsp_scale`) and struct out-parameters (`ferrum_dsp_stats`)

This sample uses the test_stub fixture to demonstrate the basic P/Invoke flow.

## What It Demonstrates

1. **Simple P/Invoke**: Calling `ferrum_add(int, int)` from C via `[LibraryImport]`
2. **Zero-copy buffers**: Using `NativeBuffer<T>` to pass pinned memory to native code
3. **NativeAOT on iOS**: The sample is configured with `PublishAot=true` for iOS
4. **Cross-platform native library integration**: XCFramework for iOS, `.so` files for Android

> **Note:** The framework also includes a `dsp_fixture` that validates float buffer processing 
> and struct out-parameters. See [docs/architecture.md](../../docs/architecture.md) for details 
> on all test fixtures and their purposes.

## Prerequisites

Before running this sample, you must build the native test fixtures for your target platform.

### iOS

Requires macOS with Xcode 15+:

```bash
# From repository root
./native/scripts/build_ios.sh
```

This produces `artifacts/ios/libferrum_test_stub.xcframework` (and `libferrum_dsp_fixture.xcframework`).

### Android

Requires Android NDK r25+ (set `ANDROID_NDK_HOME`):

```bash
# From repository root
export ANDROID_NDK_HOME=/path/to/ndk  # e.g., ~/Library/Android/sdk/ndk/25.2.9519653
./native/scripts/build_android.sh
```

This produces `.so` files for both test fixtures in `artifacts/android/jniLibs/{arm64-v8a,armeabi-v7a,x86_64}/`.

## Building and Running

### Using Visual Studio

1. Open `Ferrum.sln` in Visual Studio
2. Set `MinimalDemo` as the startup project
3. Select your target (Android Emulator, iOS Simulator, or device)
4. Press F5 or click "Run"

### Using Command Line

#### Android

```bash
# Build
dotnet build -f net9.0-android

# Deploy and run on emulator/device
dotnet build -f net9.0-android -t:Run
```

#### iOS (macOS only)

```bash
# Build
dotnet build -f net9.0-ios

# Deploy and run on simulator
dotnet build -f net9.0-ios -t:Run
```

## How It Works

This sample demonstrates the basic pattern using the `test_stub` fixture.

### 1. Native Library (`libferrum_test_stub`)

The native library is a simple C implementation:

```c
// native/test_stub/add.c
int ferrum_add(int a, int b) {
	return a + b;
}
```

### 2. C# Bindings (`AddInterop.cs`)

Generated by `ferrum-codegen`:

```csharp
[LibraryImport(InteropConstants.FerrumLibrary)]
internal static partial int ferrum_add(int a, int b);
```

Where `FerrumLibrary` is:
- `__Internal` on iOS (statically linked)
- `libferrum_test_stub` on Android (loaded from APK)

### 3. MAUI UI Call

```csharp
private void OnCallNativeClicked(object sender, EventArgs e)
{
	int result = AddInterop.ferrum_add(21, 21);
	ResultLabel.Text = $"ferrum_add(21, 21) = {result}";
}
```

### 4. Zero-Copy Buffer Demo

```csharp
using var buf = new NativeBuffer<float>(256);

// Write to managed Span
for (int i = 0; i < buf.Length; i++)
	buf.Span[i] = i * 0.1f;

// Pass pointer to native code (pinned, no copy)
unsafe { SomeNativeFunc(buf.TypedPointer, buf.Length); }
```

## Expected Output

When you tap **"Call Native Code"**:
```
ferrum_add(21, 21) = 42
```

When you tap **"Buffer Demo"**:
```
Created 256-element buffer. First 5: 0.0, 0.1, 0.2, 0.3, 0.4
```

## Troubleshooting

### iOS: `dllimport not found for __Internal`

**Cause:** Native library not built or not referenced in `.csproj`.

**Fix:**
1. Run `./native/scripts/build_ios.sh` from repo root
2. Verify `MinimalDemo.csproj` contains:
   ```xml
   <NativeReference Include="..\..\artifacts\ios\libferrum_test_stub.xcframework">
	 <Kind>Framework</Kind>
	 <SmartLink>true</SmartLink>
   </NativeReference>
   ```

### Android: `DllNotFoundException: libferrum_test_stub`

**Cause:** `.so` files not built or not included in APK.

**Fix:**
1. Set `ANDROID_NDK_HOME` and run `./native/scripts/build_android.sh`
2. Verify `MinimalDemo.csproj` includes:
   ```xml
   <AndroidNativeLibrary Include="..\..\artifacts\android\jniLibs\arm64-v8a\libferrum_test_stub.so" />
   <AndroidNativeLibrary Include="..\..\artifacts\android\jniLibs\armeabi-v7a\libferrum_test_stub.so" />
   <AndroidNativeLibrary Include="..\..\artifacts\android\jniLibs\x86_64\libferrum_test_stub.so" />
   ```

### Build fails on Linux or without MAUI workload

**Cause:** The sample project targets `net9.0-ios` and `net9.0-android` which require MAUI workloads.

**Fix:** This sample is designed for iOS/Android and cannot run on Linux. Use a macOS or Windows machine with MAUI workloads installed.
